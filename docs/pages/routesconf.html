This file defines the routes recognised by the framework, which are able to be routed to different parts of the framework. It can use placeholders so that dynamic URIs can be used, and variables can be passed to the action that the route uses. The basic syntax is as follows:

<route match="route">
  <redirect route="newroute"[ secure="secure"] />
</route>

<route match="route">
  <execute controller="controller"[ action="action"][ secure="secure"] />
</route>

In the first example, anyone visiting the URI specified by route will be redirected to the route specified by newroute. This allows you to send users elsewhere in the application without being dependent on where the application is installed. You can specify a secure attribute with the value 'yes' or 'no' to force users to https or http protocols. Omitting the attribute leaves the protocol unchanged.

Instead of specifying a new route to redirect to, you can instead specify a URI with the uri attribute, which redirects to an external URI.

In the second example, anyone visiting the URI specified by route will be directed to the Controller specified by controller and the method specified by action. If no action is specified, the index() action of the controller will be used by default.

Routes can contain placeholders to represent dynamic parts of the uri. The available placeholders are:

{int}
Represents an integer
{integer}
Alias of {int}
{string}
A string - any character except "/"
{url}
A string that can include "/" as well

Some examples of routes can be found in the comments in the routes.xml file.

If you want to change any of the information in this file, only edit the version in app/config/. The version in sys/config/ has no effect, it is only there as a backup of the default file contents should you need to revert to them.
