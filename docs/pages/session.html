Session Class

< Back

The Session class acts as a simple wrapper for Sessions within the framework. The class offers no extended functionality to sessions, other than an object oriented interface, but is used by the rest of the framework and should be used by any application code.

The idea of using this is that the behaviour or methods used by the session class can be extended or changed in the future very easily if need be. The class uses the native PHP Sessions mechanism underneath the bonnet, and should be the only way sessions are accessed. The Database Sessions functionality also makes use of this class.

This class is automatically available within Controllers as $this->session.

Methods & Properties

Session->var

You can get or set a session variable this way, check if it is set with isset() and unset it.

Session->regenerateID()

Regenerates the Session ID.

Session->destroy()

Destroys the session.

Example

$objSession = new Session();

$objSession->foo = "bar";
$objSession->xyz = 0;

echo $objSession->foo;
echo $objSession->bar;

var_dump(isset($objSession->baz));

$objSession->regenerateID();

$objSession->destroy();
