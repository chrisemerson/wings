ErrorRegistry Class

< Back

The Error Registry acts as a central place to store errors for the current request, so they can all be displayed at once, no matter what the source of the error was.

In addition, you can set the name of the field that errored, so when working with a form, you are able to tell if a particular field has errored or not.

The InputFilter class adds its errors to the ErrorRegistry when validating form input, so they can be easily displayed, and classes can be applied to the field that errored.

The Errors in this class persist over every instance of the class, so many separate components can instantiate and use this class. Controllers can access this class automatically as $this->errors.

Methods & Functions

ErrorRegistry->addError ($strMessage, $strField = '')

This adds an error message to the registry. Optionally, you can also pass in a field name, which associates the error with the named field.

ErrorRegistry->getErrors ()

Returns an array of the error messages added so far.

ErrorRegistry->getFieldErrors ($strField)

Returns an array of the error messages that have been associated with the named field.

ErrorRegistry->isError ()

Returns true if any error messages have been added so far, and false if not.

ErrorRegistry->isFieldError ($strField)

Returns true if any error messages have been associated with the named field, and false if not.

ErrorRegistry->getErroredFields ()

Returns an array containing the names of all the fields that have had errors associated with them so far.

ErrorRegistry->clearErrors ()

Clears all the errors that have been entered so far.
