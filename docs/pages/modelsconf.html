This file defines the models that are stored in the database, which will be autocreated by Wings, and can use the ORM functionality provided by the framework.

Each line consists of a <model> element with at a minimum, a name attribute, which defines the name of the model, and should be camelCased.

The rest of the information is derived from this name - the plural is simply the name with an s added to the end, and the table name is the pluralised name with the camelCase converted into a lowercase name with underscores between words.

If required, either the plural or the table name can be overridden by providing a plural or table attribute onto the <model> element. For example:

<models>
  <model name="ExampleModel" />
  <model name="AnotherDemo" table="demo_table" />
  <model name="Address" plural="addresses" />
  <model name="Mouse" plural="mice" table="mouse_table" />
</models>

If you want to change any of the information in this file, only edit the version in app/config/. The version in sys/config/ has no effect, it is only there as a backup of the default file contents should you need to revert to them.
