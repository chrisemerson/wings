Loader Class

< Back

The Loader class is responsible for interpreting the names of classes that aren't yet loaded in the framework, working out what they are and where the relevant file lives, and loading it. The __autoload() function in /sys/base/3.inc/inc.autoload.php uses this class.

The Loader class interprets the name using the framework's naming convention, with any other names being treated as either a Model, Library Class or Third Party Class in that order of priority.

This class is intended for internal use only and should not be needed by application code.

Methods & Properties

Loader->__construct($strClassName) (Constructor)

The constructor takes a single argument, which is the name of the class you are attempting to load.

Loader->load()

This method call prompts the class to search for the class in question and load it, according to the naming conventions and locations in the framework.
