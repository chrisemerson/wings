errors.xml

< Back

This file defines the default error messages used by the InputFilter class. These are displayed to the user when one of the things they enter on a form fails validation. {field} is replaced with the name of the field (as passed to the InputFilter class), and each numbered field ({n}) is replaced with the appropriately numbered parameter that is passed to the validation method.

Each element under the root element <error> is the name of the validation method to which each error text applies. Under each validation method is the <default> message for each one. You can also include a <fields> element to override the default message for individual fields, as follows:

<errors>
  <integerpositive>
    <default>{field} must be a positive integer</default>

    <fields>
      <category>You must select a Category</category>
    </fields>
  </integerpositive>

  [...]
</errors>

To override the settings, first copy the file from sys/config/ to app/config/, and modify the copy.

Default File Contents

<?xml version="1.0" encoding="utf-8"?>

<errors>
  <required>
    <default>{field} is required, and was not entered.</default>
  </required>

  <requiredif>
    <default>{field} is required, and was not entered.</default>
  </requiredif>

  <is>
    <default>{field} has an incorrect value</default>
  </is>

  <matches>
    <default>{field} must match {2}.</default>
  </matches>

  <length>
    <default>{field} must be exactly {1} characters long.</default>
  </length>

  <lengthmin>
    <default>{field} must be at least {1} characters long.</default>
  </lengthmin>

  <lengthmax>
    <default>{field} must be {1} characters or less.</default>
  </lengthmax>

  <lengthbetween>
    <default>{field} must be between {1} and {2} characters long.</default>
  </lengthbetween>

  <alpha>
    <default>{field} can only consist of alphabetical characters.</default>
  </alpha>

  <numeric>
    <default>{field} must be a numeric value.</default>
  </numeric>

  <alphanumeric>
    <default>{field} can only consist of alphanumeric characters.</default>
  </alphanumeric>

  <integer>
    <default>{field} must be an integer.</default>
  </integer>

  <integermin>
    <default>{field} must be an integer and no lower than {1}.</default>
  </integermin>

  <integermax>
    <default>{field} must be an integer and no higher than {1}.</default>
  </integermax>

  <integerbetween>
    <default>{field} must be an integer between {1} and {2}.</default>
  </integerbetween>

  <integerpositive>
    <default>{field} must be a positive integer</default>
  </integerpositive>

  <integernegative>
    <default>{field} must be a negative integer</default>
  </integernegative>

  <integernonpositive>
    <default>{field} must be a non-positive integer</default>
  </integernonpositive>

  <integernonnegative>
    <default>{field} must be a non-negative integer</default>
  </integernonnegative>

  <integernonzero>
    <default>{field} must be an integer, and must not be 0</default>
  </integernonzero>

  <regex>
    <default>{field} is in an invalid format</default>
  </regex>

  <validemail>
    <default>{field} must be a valid email address.</default>
  </validemail>

  <unique>
    <default>{field} is already taken. Please try another one!</default>
  </unique>
</errors>
