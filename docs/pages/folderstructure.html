Folder Structure

< Back

The basic folder structure of PHP Wings is shown below. The layout encourages application specific code to be separated from the framework code, in order to allow upgrading the framework to be as easy as possible. You can move the app/ and sys/ folders to anywhere on the system, as long as they are in the same folder, and you update the location in the require_once line of httpdocs/index.php.

  • / - Base Folder
    • app/ - Application specific Folder - all application code goes in here
      • cache/ - Cache folder - should be writable
      • config/ - Folder for config files, all of which should be xml. For use with the config class.
      • controllers/ - Controllers reside here. Should be called controller.<name>.php
      • cron/ - Cron task scripts go in here
      • data/ - Generic data files can go here for use by anything else - text, xml, image files, uploads etc
      • lib/ - Library, contains helper classes and modules, like Captcha, Session, Date, Page Manager etc - class.<name>.php
        • res/ - Library resources - eg Backgrounds and fonts for captcha
        • thirdparty/ - Third party classes and software, eg phpmailer, included files configured by thirdparty.xml config file
      • logs/ - Log files for the application
      • models/ - model files, model.<name>.php - eg Student, Book, Product etc.
      • templates/ - .tpl template files, for use with the template class.
        • system/ - Built in templates, used by the base system. Overridable, but shouldn't remove or rename
      • tools/ - One off tool files for, eg, clearing the database, updating a schema, gathering data etc
      • views/ - View helper files, called view.<name>.php. Master view in here, overridable, not renamable.
    • db/ - A folder for use storing database patches and dumps. Not used by the system, provided for convenience only, or can be deleted if not needed.
    • docs/ - This Folder - Documentation. Not used by the system, can be deleted if needed.
    • httpdocs/ - Publically Accessible folder - Apache should serve from here.
      • images/ - Empty folder - for use by images
      • scripts/ - Empty Folder - for use by JS
      • styles/ - Empty Folder - for use by CSS
    • sys/ - System folder - Nothing in here should be changes. All code should go in app/
      • base/ - Base application classes and files. Automatically included by the framework.
        • 1.interfaces/ - Interface files, for defining DB drivers etc
        • 2.classes/ - Base classes required by the system
        • 3.inc/ - General files to include - 'Bootstrap' files. Contains magic quotes fix, session start, __autoload() function etc
      • config/ - Folder for config files, all of which should be xml. For use with the config class.
      • controllers/ - Built in controllers. Currently only consists of Error controller
      • dbdrivers/ - DB driver files - common interfaces for different database types, db.<name>.php
      • lib/ - Library, contains helper classes and modules, like Captcha, Session, Date, Page Manager etc - class.<name>.php
        • thirdparty/ - Third party classes and software, eg phpmailer, included files configured by thirdparty.xml config file
      • templates/ - .tpl template files, for use with the template class.
        • system/ - Built in templates, used by the base system. Overridable, but shouldn't remove or rename
      • views/ - Built in Views - Default and Error views.

(foldername/ indicates that the folder needs to be writable by the web server. The data/ folder may need to be writable if your application uses it for uploads.)
