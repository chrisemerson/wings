app.xml contains most application settings, such as path information, URI information, database connection information etc.

It also has the ability to supply different settings depending on the environment - see the page on the Config class for more information on this behaviour.

If you want to change any of the information in this file, only edit the version in app/config/. The version in sys/config/ has no effect, it is only there as a backup of the default file contents should you need to revert to them.

Default File Contents

<?xml version="1.0" encoding="utf-8"?>

<app>
  <default>
    <uri>
      <host>dev</host>
      <path>appframework/httpdocs</path>
    </uri>

    <template>
      <mastertemplate>system.master</mastertemplate>
    </template>

    <db>
      <uri>mysqli://app:*****@localhost/app</uri>
      <prefix>app_</prefix>
    </db>

    <errors>
      <showerrors>0</showerrors>
      <debugmode>0</debugmode>
      <defaultformerror>{field} errored</defaultformerror>
    </errors>

    <cookie>
      <domain></domain>
      <path>/appframework/httpdocs/</path>
    </cookie>
  </default>

  <dev>
    <errors>
      <showerrors>1</showerrors>
      <debugmode>1</debugmode>
    </errors>
  </dev>
</app>
