Application Class

< Back

The Application class contains a number of static methods that could be called on from anywhere. Mostly these are helper methods and methods that return information from app.xml - anything that is useful to have globally in an application.

Methods & Properties

Application::getCurrentPageURI ($arrQSA = array())

This method returns the URI of the current page. You can optionally pass in an array of key/value pairs to be added to the query string already in the URI.

Application::getBaseURI ($blnSecure = false)

This method returns the base URI of the application - the location in which it is installed. You can pass in an parameter which determines whether the URI is retuned with the http or https protocol used.

Application::getCurrentPageURIRelativeToBase ($arrQSA = array())

This method returns the difference between the last 2 method calls - the current page URI without the base URI in front of it! As with getCurrentPageURI(), you can pass an associative array of query string additions.

Application::getFullURI ($strPath = "/", $blnSecure = false)

This method allows you to pass in a path relative to the base URI of the application, and recieve a full URI back. As with getBaseURI(), you also need to pass in an parameter which determines whether the protocal used is http or https.

Application::getBasePath ()

This returns the base path of the system, which is the folder that the sys/ and app/ folders live in. For the most part, this is also the same path that httpdocs/ is in.

Application::getAppFilename ($strFilename)

This method takes a filename (relative to the either the sys/ or app/ folder), and returns a full filename. If the file is found within the app/ folder, this filename is returned. If not, it looks in the sys/ folder instead, and returns this filename if found. If the file is in neither location, it returns false.

Application::isSecure ()

Returns true if the page is being accessed over an https connection, false otherwise.

Application::doNotTrack ()

Returns true if the user has checked the 'Do Not Track' option in their browser, false otherwise.

Application::redirect ($strURL)

Redirects the user to the given URI.

Application::getEnvironment ()

Returns the current environment, as set by the APP_ENVIRONMENT server environment variable, or false if it is not set.

Application::exitApp ($strMessage = "")

Terminates processing of the application and stops output. This makes sure the session information is saved before calling PHP's exit() function. You can optionally pass in a message which is output before the execution stops.

Application::showError ($strErrorType, $mixErrorContent = '', $strLoggedError = '')

Shows an error page, depending on what was passed in. Calls the showErrorTypeError action in the ErrorController. For example, calling Application::showError('404') calls the ErrorController->show404Error() action. The Error Content parameter is passed as a parameter to the action. The Logged Error parameter currently does nothing.

Application::handleUncaughtException ($exException)

This is used internally as an exception handler to show an Error Page for uncaught exceptions, including display of the debug backtrace.
